# Multi-Agent Architectures Research

Этот репозиторий исследует различные способы взаимодействия LLM-агентов: от строгих корпоративных структур до биологически-подобных нейронных связей.

| Проект / Папка | Тип системы | Статус | Главный плюс | Главный минус |
| :--- | :--- | :--- | :--- | :--- |
| **NeoN** | Иерархическая (Корпорация) | Стабильно | Всестороннее обсуждение | Глупые ошибки менеджера, "размытые" ответы |
| **analys_core** | Распределенно-иерархическая | В разработке | Устойчивость к ошибкам формата | Меньше контекста у спецов, крайне неэффективный менеджмент |
| **experimental_core** | Доска объявлений | Не работает | Красивая концепция | Абсолютно неработоспособна |
| **brain** | Инновационная (Нейроны) | Эксперимент | Перспективно (как мозг) | Крайне нестабильно, сложно в отладке |

## Основные идеи по улучшению:
Динамический контекст из единой векторной базы знаний (как история диалога, так и общие факты)
Распределенная сборка ответа: идея решена -> записано в ответ
Блок инноваций: попытка применения нескольких фактов из разных областей в рамках решения задачи
Самоанализ: мета-агенты могут находить ошибки и генерировать дообучающие данные, а так же использовать саму систему для улучшения системы
Эксперименты: если нет нужных данных - нужно их получить. Агенты должны предлагать способ получить необходимые данные, а лучше и самим проводить эксперименты
Узкая специализация агентов: каждый эксперт может быть одной легкой, узкоспециализированной моделью, например VibeThinker (https://huggingface.co/WeiboAI/VibeThinker-1.5B) - математик
